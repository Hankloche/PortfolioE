  • Résolution du problème de VM avec Kali Linux et tests des fonctionnalités PGP :
    • Utilisation de Kali Linux pour résoudre le problème : Mise en place d’une machine virtuelle sous Kali Linux pour contourner les difficultés rencontrées avec Debian 12.       • gpg --card-status : Affiche les informations détaillées sur une carte intelligente utilisée avec GPG, comme une YubiKey. Permet de confirmer que la carte est détectée correctement et prête à être utilisée.
    • Tests sur VMware et VirtualBox : Déploiement de Kali Linux sur deux environnements virtuels : VMware et VirtualBox. Comparaison des performances et des capacités de gestion des périphériques USB, comme la YubiKey, dans les deux hyperviseurs.

    • Problème d’utilisation de deux VMs simultanément : Les deux machines virtuelles (VM) utilisées en parallèle (VMware et VirtualBox) causaient des conflits, notamment lors de l’accès à la YubiKey en USB partagé. 21/01/2025

  • Compte-rendu des tests pour le hook Git avec signature GPG :
    • Objectif : Mettre en place un hook Git pour empêcher les commits non signés avec une clé GPG.
    • Étapes réalisées :
      • Commandes testées : Utilisation de la commande suivante dans le script du hook : git commit --dry-run
        Observation : Cette commande simule le commit mais ne fournit aucune information sur la signature GPG, rendant son utilisation inefficace pour détecter les commits non signés.
      • Autres essais : Expérimentation avec des hooks Git :
        • commit-msg
        • pre-commit
        Observation : Ces approches n’ont pas permis de détecter ou de forcer la signature des commits avec une clé GPG.
    • Résultat : Aucun des scripts ou méthodes testés jusqu’à présent n’a permis de bloquer efficacement les commits non signés.
    • Prochaine étape :
      • Recherche d’une méthode ou commande efficace pour vérifier la signature GPG des commits avant leur autorisation.
      • Exploration de pistes potentielles, comme l’utilisation de la commande : git log --show-signature
